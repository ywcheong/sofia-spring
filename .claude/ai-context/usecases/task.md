## UC-003: 번역 과제 생성 (Create Translation Task)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-005, US-006, US-007, US-008

**전제 조건(Preconditions)**:
- 시스템이 번역 페이즈 상태

**기본 흐름(Main Flow)**:
1. 관리자가 과제 유형 선택: 가온누리 게시글 또는 외부 게시글
2. 관리자가 게시글 유형 및 Work ID 입력
	- 가온누리 게시글: 문자열 ID (게시글 번호)
	- 외부 게시글: 과제명 (최대 50자)
3. 시스템이 유형 및 Work ID 중복 여부 확인
4. [중복 시] 오류 메시지 표시 후 종료
5. 관리자가 배정 방식 선택: **자동 배정** 또는 **수동 배정**

**자동 배정 흐름(자동 배정 선택 시)**:
6a. 시스템이 순환 알고리즘으로 할당 대상 번역버디 미리보기 표시
- 휴식 상태 번역버디 제외
- 순환(라운드 로빈) 방식으로 할당
- 동시에 여러 관리자가 과제 생성 시 경합 발생 가능 (DB 트랜잭션으로 처리)
  7a. 관리자가 배정 대상 확인 후 과제 생성 요청
  8a. 시스템이 과제를 선택된 번역버디에게 할당
  9a. 할당 시각 기록

**수동 배정 흐름(수동 배정 선택 시)**:
6b. 관리자가 드롭다운에서 할당할 번역버디 선택
- 휴식 상태 번역버디는 선택지에서 제외
  7b. 관리자가 과제 생성 요청
  8b. 시스템이 과제를 선택된 번역버디에게 할당
  9b. 할당 시각 기록

**공통 흐름(이메일 발송)**:
10. 시스템이 이메일 발송 안내 표시: "과제 할당 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
	- 이메일 수신거부 시 발송 생략
11. 관리자가 확인 클릭
12. [수신 허용 시] 해당 번역버디에게 과제 할당 알림 이메일 발송
13. [수신거부 시] 관리자에게 "해당 사용자는 이메일 수신을 거부했습니다" 메시지 표시

**종료 조건(Postconditions)**:
- 과제가 시스템에 생성됨
- 과제가 휴식 상태가 아닌 번역버디에게 할당됨
- 할당 시각이 기록됨
- 번역버디에게 알림 발송됨 (수신 허용 시)

## UC-004: 과제 완료 보고 (Report Task Completion)

**행위자(Actor)**: 번역버디

**관련 사용자 스토리**: US-011

**전제 조건(Preconditions)**:
- 사용자가 승인된 참가자 상태
- 할당된 미완료 과제가 존재
- 사용자가 카카오톡 챗봇과 대화 중

**기본 흐름(Main Flow)**:
1. 번역버디가 할당된 과제 목록 조회
2. 번역버디가 완료할 과제 선택
3. 번역버디가 번역 글자 수(공백 제외) 입력
4. 번역버디가 완료 보고 제출
5. 시스템이 현재 시각과 할당 시각 비교
6. [48시간 초과 시] 자동 경고 발행
7. 시스템이 봉사시간 계산: `글자 수 × 3.942초` (표시용)
8. 참가자의 총 번역 글자 수에 글자 수 추가 (저장)
9. 과제 완료 상태로 변경
10. 완료 시각 기록
11. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 과제가 완료 상태로 변경됨
- 참가자의 총 번역 글자 수가 업데이트됨
- [지각 시] 해당 참가자에게 경고가 추가되고 경고 알림 이메일이 발송됨 (이메일 수신 허용 시)

## UC-005: 내 통계 확인 (View My Statistics)

**행위자(Actor)**: 번역버디

**관련 사용자 스토리**: US-010, US-012

**전제 조건(Preconditions)**:
- 사용자가 승인된 참가자 상태
- 사용자가 카카오톡 챗봇과 대화 중

**기본 흐름(Main Flow)**:
1. 번역버디가 통계 조회 요청
2. 시스템이 참가자 정보 조회
3. 시스템이 다음 정보 표시:
	- 학번, 이름
	- 경고 횟수
	- 총 번역 글자 수
	- 예상 봉사시간 (글자 수 × 3.942초 + 총 보정 자수 × 3.942초)
	- 완료한 과제 목록
	- 진행 중인 과제 목록

**종료 조건(Postconditions)**:
- 참가자 통계가 표시됨

## UC-013: 개인 휴식 설정 (Set Individual Rest Status)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-009

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근
- 휴식 상태를 설정할 번역버디가 존재

**기본 흐름(Main Flow)**:
1. 관리자가 번역버디 목록 조회
2. 관리자가 특정 번역버디 선택
3. 관리자가 휴식 상태 설정 또는 해제 선택
4. 시스템이 번역버디의 휴식 상태 업데이트

**종료 조건(Postconditions)**:
- 휴식 상태 설정 시: 해당 번역버디는 과제 할당에서 제외됨
- 휴식 상태 해제 시: 해당 번역버디는 과제 할당 대상에 포함됨
