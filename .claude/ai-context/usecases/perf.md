# 경고 및 봉사시간 관리 유스케이스 명세

## 관련 사용자 스토리

| ID | 행위자 | 내용 | 가치/이유 |
|----|-------|------|---------|
| US-013 | 시스템 | 48시간 이상 지연된 제출에 자동 경고를 발행하고 싶다 | 제출 기한을 준수하도록 유도하기 위해서 |
| US-014 | 관리자 | 번역버디에게 경고를 수동으로 부여/취소하고 싶다 | 규정 위반 시 제재를 부과하기 위해서 |

## UC-006: 경고 발행/취소 (Issue/Cancel Warning)

**행위자(Actor)**: 관리자 (수동), 시스템 (자동)

**관련 사용자 스토리**: US-013, US-014

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근 (수동 경고 시)
- 경고를 발행하거나 취소할 번역버디가 존재

**기본 흐름(Main Flow)**:
1. 관리자가 번역버디 목록 조회
2. 관리자가 특정 번역버디 선택
3. 관리자가 경고 발행 또는 취소 선택

**경고 발행 흐름**:
4a. 관리자가 경고 사유 입력
5a. 시스템이 이메일 발송 안내 표시: "경고 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
6a. 관리자가 확인 클릭
7a. 시스템이 해당 번역버디의 경고 횟수 증가
8a. [수신 허용 시] 번역버디에게 경고 알림 이메일 발송
8b. [수신거부 시] 관리자에게 "해당 사용자는 이메일 수신을 거부했습니다" 메시지 표시
9a. 성공 메시지 표시

**경고 취소 흐름**:
4b. 관리자가 취소할 경고 선택
5b. 시스템이 해당 번역버디의 경고 횟수 감소
6b. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 경고 발행 시: 번역버디 경고 횟수가 증가
- 경고 취소 시: 번역버디 경고 횟수가 감소

## UC-007: 봉사시간 수동 조정 (Adjust Volunteer Hours)

> **참고**: 이 유스케이스는 실제로 보정 자수를 조정하며, 봉사시간은 파생 필드로 계산됩니다.

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-015, US-016

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근
- 보정할 참가자가 존재

**기본 흐름(Main Flow)**:
1. 관리자가 참가자 목록 조회
2. 관리자가 특정 참가자 선택
3. 관리자가 조정 자수 입력 (양수: 가산, 음수: 차감)
4. 관리자가 조정 사유 입력
5. 시스템이 보정 기록 저장
	- 대상 사용자 고유 식별자
	- 조정 자수
	- 사유
	- 등록 시각
	- 등록자 고유 식별자
6. 참가자의 총 보정 자수 업데이트
7. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 보정 기록이 시스템에 저장됨
- 참가자의 총 보정 자수가 업데이트됨
