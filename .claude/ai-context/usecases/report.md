## UC-011: 성과 보고서 생성 (Generate Performance Report)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-020

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근
- 시스템이 모집 페이즈 이후 상태

**기본 흐름(Main Flow)**:
1. 관리자가 보고서 생성 요청
2. 시스템이 모든 참가자 데이터 수집
3. 시스템이 다음 정보를 포함하는 CSV 생성:
	- 학번
	- 이름
	- 총 번역 글자 수
	- 예상 봉사시간 (글자 수 × 3.942초 + 총 보정 자수 × 3.942초)
	- 경고 횟수
	- 총 보정 자수
4. CSV 파일 다운로드 제공

**종료 조건(Postconditions)**:
- 성과 보고서 CSV 파일이 생성됨

## UC-012: 페이즈 전환 (Change System Phase)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-021

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근

**기본 흐름(Main Flow)**:
1. 관리자가 페이즈 전환 요청
2. 관리자가 새로운 페이즈 선택
	- 비활성: 모든 기능 중단
	- 모집: 신청, 승인, 팀 구성 가능
	- 번역: 위 기능 + 과제 생성, 완료 보고 가능
	- 정산: 신규 과제 생성 불가, 보고서 생성 가능
3. [페이즈별 사전 조건 검증]
	- 번역 → 정산: 모든 과제가 완료 상태여야 함
	- 모집 → 번역: 승인 대기 중인 참가자가 없어야 함
4. 관리자가 유지 대상 선택: "관리자만 유지" 또는 "모든 참가자 유지"
5. 관리자가 전환 확인
6. 시스템이 페이즈 변경
7. [선택에 따라] 비관리자 참가자 상태 초기화 또는 유지
8. 페이즈 시작일 기록
9. 해당 페이즈에 맞는 기능 활성화/비활성화
10. 전환 로그 기록

**종료 조건(Postconditions)**:
- 시스템 페이즈가 변경됨
- 페이즈별 허용 기능이 적용됨
- 전환 로그가 기록됨 (US-022)
