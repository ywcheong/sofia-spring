# 참가 신청 및 승인 유스케이스 명세

## 관련 사용자 스토리

| ID | 행위자 | 내용 | 가치/이유 |
|----|-------|------|---------|
| US-001 | 번역버디 | 봉사활동에 참여 신청을 하고 싶다 | 봉사시간을 쌓기 위해서 |
| US-002 | 관리자 | 신청한 참가자를 검토하여 승인/거절하고 싶다 | 적격한 참가자만 선별하기 위해서 |

---

## UC-001: 참가 신청 (Apply for Participation)

**행위자(Actor)**: 번역버디

**관련 사용자 스토리**: US-001

**전제 조건(Preconditions)**:
- 시스템이 모집 또는 번역 페이즈 상태

**기본 흐름(Main Flow)**:
1. 번역버디가 학번(XX-XXX)과 이름(2~4글자)을 입력하여 신청 제출
2. 시스템이 학번 중복 여부 확인
3. [중복 시] 오류 메시지 표시 후 종료
4. 시스템이 학번 형식과 이름 길이 검증
5. [형식 불일치 시] 오류 메시지 표시 후 종료
6. 시스템이 승인 대기 상태의 신청을 저장
7. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 신청 정보가 시스템에 저장됨
- 신청 상태가 "승인 대기"로 설정됨

## UC-002: 신청 승인/거절 (Approve/Reject Application)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-002

**전제 조건(Preconditions)**:
- 시스템이 모집 또는 번역 페이즈 상태
- 대기 중인 신청이 존재

**기본 흐름(Main Flow)**:
1. 관리자가 대기 중인 신청 목록 조회
2. 관리자가 특정 신청 선택
3. 관리자가 승인 또는 거절 선택

**승인 흐름(승인 선택 시)**:
4a. 시스템이 해당 신청자를 승인 상태로 변경
5a. 시스템이 이메일 발송 안내 표시: "승인 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
6a. 관리자가 확인 클릭
7a. 참가자에게 승인 알림 이메일 발송

**거절 흐름(거절 선택 시)**:
4b. 사유 입력 (선택)
5b. 시스템이 이메일 발송 안내 표시: "거절 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
6b. 관리자가 확인 클릭
7b. 신청 상태가 "거절"로 변경됨
8b. 참가자에게 거절 알림 이메일 발송

**종료 조건(Postconditions)**:
- 승인 시: 참가자의 상태가 "승인"으로 변경
- 거절 시: 상태가 "거절"로 변경
