|----|-------|------|---------|
| US-026 | 시스템 | 과제가 할당될 때 번역버디에게 이메일을 발송하고 싶다 | 과제 할당 사실을 알리기 위해서 |

## UC-019: 이메일 수신 설정 (Configure Email Reception)

**행위자(Actor)**: 번역버디

**관련 사용자 스토리**: US-030

**전제 조건(Preconditions)**:
- 사용자가 승인된 참가자 상태

**기본 흐름(Main Flow)**:
1. 번역버디가 이메일 하단의 "수신거부" 링크 클릭
2. [관리자인 경우] 시스템이 해당 사용자의 인증 토큰을 통해 본인 확인
3. [일반 번역버디인 경우] 시스템이 이메일에 임베디된 이메일_인증_비밀키와 이메일에 포함된 키가 매칭되는지 확인
4. [인증 성공 시] 시스템이 이메일 수신 여부 토글
5. [수신거부 시] 이메일 수신이 중단됨
6. [수신 허용 시] 이메일 수신이 재개됨
7. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 번역버디의 이메일 수신 여부가 변경됨
- 수신거부 시: 이후 이메일이 발송되지 않음
- 수신 허용 시: 이후 이메일이 정상 발송됨

## UC-020: 이메일 발송 (Send Email)

**행위자(Actor)**: 시스템

**관련 사용자 스토리**: US-026, US-027, US-028, US-029, US-031

**전제 조건(Preconditions)**:
- 이메일 발송 요청이 존재

**기본 흐름(Main Flow)**:
1. 시스템이 이메일 수신 여부 확인
2. [수신거부 시] 이메일 발송 건너뜀, 관리자에게 "해당 사용자는 이메일 수신을 거부했습니다" 메시지 표시
3. [수신 허용 시] 이메일 발송 진행
4. [수신 허용 시] 모든 이메일 하단에 "수신거부" 링크 포함
5. 발송 상태 기록

**발송 트리거 유형**:
| 트리거 | 발송 대상 | 관련 US |
|-------|---------|--------|
| 과제 할당 | 해당 번역버디 | US-026 |
| 경고 발행 | 해당 번역버디 | US-027 |
| 신청 승인/거절 | 신청자 | US-028 |
| 관리자 승급/강등 | 해당 참가자 | US-029 |

**종료 조건(Postconditions)**:
- 수신거부 시: 이메일이 발송되지 않고 관리자에게 알림 표시
- 수신 허용 시: 이메일이 정상 발송되고 "수신거부" 링크 포함

## UC-021: 48시간 제출 리마인드 (48-Hour Submission Reminder)

**행위자(Actor)**: 시스템

**관련 사용자 스토리**: US-032

**전제 조건(Preconditions)**:
- 할당된 과제가 존재
- 과제가 완료되지 않은 상태
- 과제 할당 후 48시간(172,800초) 경과

**기본 흐름(Main Flow)**:
1. 시스템이 주기적으로 완료되지 않은 과제 검사
2. [48시간 경과 시] 시스템이 해당 과제 식별
3. 시스템이 이메일 수신 여부 확인
4. [수신 허용 시] 해당 번역버디에게 리마인드 이메일 발송: "과제가 할당된 지 48시간이 경과되었습니다. 제출해 주세요."
5. 모든 관리자에게 리마인드 알림 이메일 발송: "[번역버디 이름]의 과제가 48시간 경과 미제출 상태입니다."
6. 리마인드 발송 기록 저장

**종료 조건(Postconditions)**:
- 해당 번역버디에게 리마인드 이메일 발송됨 (수신 허용 시)
- 모든 관리자에게 리마인드 알림 이메일 발송됨
- 리마인드 발송 기록이 저장됨
