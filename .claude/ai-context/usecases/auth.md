## UC-016: 인증 토큰 발급 (Issue Authentication Token)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-023

**전제 조건(Preconditions)**:
- 사용자가 관리자 권한을 보유
- 사용자가 카카오톡 챗봇과 대화 중

**기본 흐름(Main Flow)**:
1. 관리자가 카카오톡 챗봇에서 토큰 발급 요청
2. 시스템이 사용자의 관리자 권한 확인
3. 시스템이 인증 토큰 생성
4. 시스템이 토큰 발급일 기록
5. 시스템이 토큰 만료일 설정 (발급일 포함 6개월)
6. 카카오톡 챗봇을 통해 토큰 전송

**종료 조건(Postconditions)**:
- 인증 토큰이 생성됨
- 토큰이 관리자에게 전송됨
- 토큰 발급일과 만료일이 기록됨

## UC-017: 인증 토큰 재발급 (Re-issue Authentication Token)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-024

**전제 조건(Preconditions)**:
- 사용자가 관리자 권한을 보유
- 사용자가 카카오톡 챗봇과 대화 중
- 기존 토큰이 존재

**기본 흐름(Main Flow)**:
1. 관리자가 카카오톡 챗봇에서 토큰 재발급 요청
2. 시스템이 사용자의 관리자 권한 확인
3. 시스템이 기존 토큰을 무효화
4. 시스템이 새로운 인증 토큰 생성
5. 시스템이 토큰 재발급일 기록
6. 시스템이 토큰 만료일 설정 (재발급일 포함 6개월)
7. 카카오톡 챗봇을 통해 새 토큰 전송

**종료 조건(Postconditions)**:
- 기존 토큰이 무효화됨
- 새로운 인증 토큰이 생성됨
- 새 토큰이 관리자에게 전송됨
- 토큰 재발급일이 기록됨

## UC-018: 웹 로그인 (Web Login)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-025

**전제 조건(Preconditions)**:
- 사용자가 웹 인터페이스 접근
- 사용자가 인증 토큰을 보유

**기본 흐름(Main Flow)**:
1. 관리자가 웹 인터페이스 접근
2. 관리자가 인증 토큰 입력
3. 시스템이 토큰 유효성 검증
4. [토큰 유효 시] 웹 인터페이스 접근 허용
5. [토큰 만료 시] 오류 메시지 표시: "토큰을 다시 발급받으세요"
6. [토큰 무효 시] 오류 메시지 표시: "유효하지 않은 토큰입니다"

**종료 조건(Postconditions)**:
- 토큰 유효 시: 웹 인터페이스 접근 허용됨
- 토큰 만료 시: 재발급 안내 메시지 표시됨
- 토큰 무효 시: 오류 메시지 표시됨
