# 관리자 권한 관리 유스케이스 명세

## 관련 사용자 스토리

| ID | 행위자 | 내용 | 가치/이유 |
|----|-------|------|---------|
| US-003 | 관리자 | 참가자를 관리자로 승급하고 싶다 | 시스템 관리 업무를 분담하기 위해서 |
| US-004 | 관리자 | 관리자 승급/강등 전 이메일 내용을 미리 확인하고 싶다 | 발송할 알림을 인지하기 위해서 |

## UC-014: 관리자 승급 (Promote to Manager)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-003, US-004, US-029

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근
- 승급할 참가자가 존재

**기본 흐름(Main Flow)**:
1. 관리자가 참가자 목록 조회
2. 관리자가 특정 참가자 선택
3. 관리자가 관리자 권한 부여 요청
4. 시스템이 이메일 발송 안내 표시: "관리자 승급 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
5. 관리자가 확인 클릭
6. 시스템이 해당 참가자의 관리자 여부를 true로 변경
7. 승급 로그 기록 (대상 사용자, 승급자, 시각)
8. 해당 참가자에게 관리자 승급 알림 이메일 발송
9. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 참가자가 관리자 권한을 획득함
- 승급 로그가 기록됨
- 이메일이 발송됨

## UC-015: 관리자 강등 (Demote from Manager)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**: US-004, US-029

**전제 조건(Preconditions)**:
- 관리자가 웹 인터페이스에 유효한 인증 토큰으로 접근
- 강등할 관리자 참가자가 존재

**기본 흐름(Main Flow)**:
1. 관리자가 참가자 목록 조회
2. 관리자가 특정 관리자 참가자 선택
3. 관리자가 관리자 권한 회수 요청
4. 시스템이 이메일 발송 안내 표시: "관리자 강등 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
5. 관리자가 확인 클릭
6. 시스템이 해당 참가자의 관리자 여부를 false로 변경
7. 강등 로그 기록 (대상 사용자, 강등자, 시각)
8. 해당 참가자에게 관리자 강등 알림 이메일 발송
9. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 참가자가 관리자 권한을 상실함
- 강등 로그가 기록됨
- 이메일이 발송됨
