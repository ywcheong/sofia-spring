# 비즈니스 요구사항 요약

## 1. 시스템 개요

### 1.1 시스템 인터페이스 구조

시스템은 두 가지 인터페이스로 구성됨.

- **카카오톡 챗봇**:
	- 참가 신청, 과제 확인, 번역 완료 보고, 통계 조회, 사전 조회, 인증 토큰 발급 요청 등 수행
- **웹 인터페이스**:
	- 승인/거절, 과제 생성, 경고 발행/취소, 보정 자수 조정, 사전 수정, 보고서 생성, 페이즈 전환,
	  개인 휴식 설정, 관리자 승급/강등 등 수행
	- 사용하기 위해서는 카카오톡 챗봇에서 발급한 인증 토큰이 필요

### 1.2 시스템 페이즈 체계

시스템은 다음 4단계의 페이즈로 운영됨. (비활성 → 모집 → 번역 → 정산) → (비활성...) 순서로 진행됨.

- **비활성**: 시스템 준비 단계로, 데이터 초기화 준비, 운영 일시 중단
- **모집**: 참가자 모집 단계로, 신청서 접수, 승인/거절 처리, 팀 구성
- **번역**: 번역 과제 수행 단계로, 과제 생성, 자동 할당, 번역 완료 보고
- **정산**: 최종 정산 단계로, 과제 생성 불가, 최종 보고서 작성, 봉사시간 확정

### 1.3 사용자 역할

- **번역버디(Translation Buddy)**
	- 봉사활동에 참여하는 학생
	- 신청, 과제 수행, 봉사시간 확인, 인증 토큰 발급 요청
- **관리자(Manager)**
	- 시스템을 관리하는 학생 (번역버디의 일부)
	- 승인, 과제 생성, 보고서 작성, 전체 설정, 기간 관리
