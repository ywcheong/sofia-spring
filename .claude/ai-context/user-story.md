## 2. 사용자 스토리

`US-### | [행위자]로서 | ~하고 싶다 [목적] | ~하기 위해서 [가치/이유]`
본 문서의 사용자 스토리는 위 형식을 따름.

### 2.1 참가 신청 및 승인

| US-001 | **번역버디** | 봉사활동에 참여 신청을 **하고 싶다** | 봉사시간을 쌓기 **위해서** |
| US-002 | **관리자** | 신청한 참가자를 검토하여 승인/거절하고 **싶다** | 적격한 참가자만 선별하기 **위해서** |

### 2.2 관리자 권한 관리

| US-003 | **관리자** | 참가자를 관리자로 승급/강등하고 **싶다** | 시스템 관리 업무를 분담하기 **위해서** |
| US-004 | **관리자** | 관리자 승급/강등 전 이메일 내용을 미리 확인하고 **싶다** | 발송할 알림을 인지하기 **위해서** |

### 2.3 과제 관리 및 수행

| US-005 | **관리자** | 새로운 번역 과제를 생성하고 **싶다** | 참가자에게 번역 작업을 할당하기 **위해서** |
| US-006 | **관리자** | 가온누리 게시글 과제와 외부 게시글 과제를 구분하고 **싶다** | 과제의 성격을 명확히 하기 **위해서** |
| US-007 | **시스템** | 과제를 번역버디에게 자동으로 할당하고 **싶다** | 관리 부담을 줄이기 **위해서** |
| US-008 | **관리자** | 과제를 특정 번역버디에게 수동으로 할당하고 **싶다** | 특별한 사정이나 요청이 있을 때 대응하기 **위해서** |
| US-009 | **관리자** | 번역버디를 휴식 상태로 설정/해제하고 **싶다** | 휴가 중인 번역버디에게 과제가 할당되는 것을 방지하기 **위해서** |
| US-010 | **번역버디** | 나에게 할당된 과제를 확인하고 **싶다** | 어떤 번역을 해야 하는지 알기 **위해서** |
| US-011 | **번역버디** | 번역 완료를 보고하고 **싶다** | 내 번역 글자 수가 누적되기를 **위해서** |
| US-012 | **번역버디** | 나의 통계와 예상 봉사시간을 확인하고 **싶다** | 현재 성과를 파악하기 **위해서** |

### 2.4 경고 및 봉사시간

| US-013 | **시스템** | 48시간 이상 지연된 제출에 자동 경고를 발행하고 **싶다** | 제출 기한을 준수하도록 유도하기 **위해서** |
| US-014 | **관리자** | 번역버디에게 경고를 수동으로 부여/취소하고 **싶다** | 규정 위반 시 제재를 부과하기 **위해서** |
| US-015 | **시스템** | 번역 글자 수로 봉사시간을 자동 계산하고 **싶다** | 계산 오류를 방지하기 **위해서** |
| US-016 | **관리자** | 특별 사유로 보정 자수를 수동으로 조정하고 **싶다** | 예외 상황을 반영하기 **위해서** |

### 2.5 번역 가이드라인

| US-017 | **번역버디** | 번역 가이드라인(사전)을 조회하고 **싶다** | 용어를 일관되게 번역하기 **위해서** |
| US-018 | **관리자** | 번역 가이드라인을 수정하고 **싶다** | 번역 품질을 유지하기 **위해서** |
| US-019 | **번역버디/관리자** | 원본 텍스트를 붙여넣어 사전에 자동으로 매핑된 치환 가능 단어들을 필터링해서 보고 **싶다** | 효율적으로 번역하기 **위해서** |

### 2.6 보고 및 관리

| US-020 | **관리자** | 성과 보고서를 CSV로 내보내고 **싶다** | 최종 봉사시간을 확정하기 **위해서** |
| US-021 | **관리자** | 운영 페이즈를 전환하고 **싶다** | 적절한 시기에 기능을 제어하기 **위해서** |
| US-022 | **시스템** | 모든 관리 작업을 기록하고 **싶다** | 감사와 책임 소재 파악을 **위해서** |

### 2.7 인증 및 토큰 관리

| US-023 | **관리자** | 웹 인터페이스 로그인을 위한 인증 토큰을 발급받고 **싶다** | 관리 기능을 이용하기 **위해서** |
| US-024 | **관리자** | 토큰 유출 시 카카오톡 챗봇을 통해 토큰을 재발급 요청하고 **싶다** | 보안을 유지하기 **위해서** |
| US-025 | **시스템** | 만료된 토큰으로 로그인 시도할 경우 "토큰을 다시 발급받으세요" 메시지를 표시하고 **싶다** | 사용자를 올바른 절차로 안내하기 **위해서** |

### 2.8 이메일 알림 및 수신 관리

| US-026 | **시스템** | 과제가 할당될 때 번역버디에게 이메일을 발송하고 **싶다** | 과제 할당 사실을 알리기 **위해서** |
| US-027 | **시스템** | 경고가 발행될 때 해당 번역버디에게 이메일을 발송하고 **싶다** | 경고 사실을 명확히 알리기 **위해서** |
| US-028 | **시스템** | 신청이 승인/거절될 때 참가자에게 이메일을 발송하고 **싶다** | 처리 결과를 알리기 **위해서** |
| US-029 | **시스템** | 관리자로 승급/강등될 때 해당 사용자에게 이메일을 발송하고 **싶다** | 권한 변경을 알리기 **위해서** |
| US-030 | **번역버디** | 이메일 수신 여부를 설정하고 **싶다** | 불필요한 알림을 차단하기 **위해서** |
| US-031 | **시스템** | 수신거부한 번역버디에게는 메일을 발송하지 않고 **싶다** | 사용자 의사를 존중하기 **위해서** |
| US-032 | **시스템** | 번역버디가 업무 할당 후 48시간 내에 제출하지 않았다면 리마인드하고 **싶다** | 제출을 유도하기 **위해서** |
