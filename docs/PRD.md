# 소피아 챗봇 - 비즈니스 요구사항 정의서
## (Sofia Chatbot - Business Requirements Document)

---

## 1. 개요 (Overview)

### 1.1 프로젝트 목적
한국과학영재학교(KSA) 국제부의 번역 봉사활동 관리 시스템으로, 학생 참가자에게 번역 과제를 자동으로 할당하고 번역 성과를 추적하여 봉사시간을 산출하는 시스템입니다.

### 1.1.1 시스템 인터페이스 구조
시스템은 두 가지 인터페이스로 구성됩니다:

| 인터페이스 | 사용자 | 주요 기능 | 비고 |
|-----------|--------|-----------|------|
| 카카오톡 챗봇 | 번역버디 (일반 및 관리자 포함) | 참가 신청, 과제 확인, 번역 완료 보고, 통계 조회, 사전 조회, 인증 토큰 발급 요청 | 번역버디의 모든 액션은 챗봇을 통해서만 수행됨 |
| 웹 인터페이스 | 관리자만 | 승인/거절, 과제 생성, 경고 발행/취소, 보정 자수 조정, 사전 수정, 보고서 생성, 기간 전환, 개인 휴식 설정, 관리자 승급/강등 | 인증 토큰 필요 |

**예외**: 번역 가이드라인(사전) 조회는 카카오톡과 웹 양쪽 모두에서 제공됩니다.

### 1.2 운영 기간 체계
시스템은 다음 4단계의 기간으로 운영됩니다:

| 기간 단계 | 설명 | 주요 활동 |
|-----------|------|-----------|
| 비활성 기간(Phase 0) | 시스템 준비 단계 | 데이터 초기화 준비, 운영 일시 중단 |
| 모집 기간(Phase 1) | 참가자 모집 | 신청서 접수, 승인/거절 처리, 팀 구성 |
| 번역 기간(Phase 2) | 번역 과제 수행 | 과제 생성, 자동 할당, 번역 완료 보고 |
| 정산 기간(Phase 3) | 최종 산출 | 신규 과제 생성 불가, 최종 보고서 작성, 봉사시간 확정 |

### 1.3 사용자 스토리 포맷
본 문서의 사용자 스토리는 다음 형식을 따릅니다:
```
[행위자]로서
~하고 싶다 [목적],
~하기 위해서 [가치/이유]
```

## 2. 사용자 역할 (User Roles)

| 역할 | 설명 | 사용 인터페이스 | 주요 활동 |
|------|------|--------------|-----------|
| 번역버디(Translation Buddy) | 봉사활동에 참여하는 학생 (일반 및 관리자 포함) | 카카오톡 챗봇 (주요), 웹 (사전 조회만, 예외적 허용) | 신청, 과제 수행, 봉사시간 확인, 인증 토큰 발급 요청 |
| 관리자(Manager) | 시스템을 관리하는 학생회원 (번역버디의 일부) | 웹 인터페이스 (인증 토큰 필요), 카카오톡 (토큰 발급/재발급 요청) | 승인, 과제 생성, 보고서 작성, 전체 설정, 기간 관리 |

**참고**: 번역버디의 모든 유스케이스는 카카오톡 챗봇을 통해서만 이루어집니다. 관리자 유스케이스는 웹 인터페이스를 통해서만 이루어집니다. 단, 번역 가이드라인(사전) 조회는 카카오톡과 웹 양쪽 모두에서 제공됩니다.

## 4. 유스케이스 명세서 (Use Case Specifications)

### UC-001: 참가 신청 (Apply for Participation)

**행위자(Actor)**: 번역버디

**관련 사용자 스토리**:
> US-001: **번역버디**로서, 봉사활동에 참여 신청을 **하고 싶다**, 봉사시간을 쌓기 **위해서**

**전제 조건(Preconditions)**:
- 시스템이 모집 기간(Phase 1) 또는 번역 기간(Phase 2) 상태

**기본 흐름(Main Flow)**:
1. 번역버디이 학번(XX-XXX)과 이름(2~4글자)을 입력하여 신청 제출
2. 시스템이 학번 중복 여부 확인
3. [중복 시] 오류 메시지 표시 후 종료
4. 시스템이 학번 형식과 이름 길이 검증
5. [형식 불일치 시] 오류 메시지 표시 후 종료
6. 시스템이 승인 대기 상태의 신청을 저장
7. 성공 메시지 표시

**종료 조건(Postconditions)**:
- 신청 정보가 시스템에 저장됨
- 신청 상태가 "승인 대기"로 설정됨

---

### UC-002: 신청 승인/거절 (Approve/Reject Application)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**:
> US-002: **관리자**로서, 신청한 참가자를 검토하여 승인/거절하고 **싶다**, 적격한 참가자만 선별하기 **위해서**

**전제 조건(Preconditions)**:
- 시스템이 모집 기간(Phase 1) 또는 번역 기간(Phase 2) 상태
- 대기 중인 신청이 존재

**기본 흐름(Main Flow)**:
1. 관리자가 대기 중인 신청 목록 조회
2. 관리자가 특정 신청 선택
3. 관리자가 승인 또는 거절 선택

**승인 흐름(승인 선택 시)**:
4a. 시스템이 해당 신청자를 승인 상태로 변경
5a. 신청 상태가 "승인"으로 변경됨
6a. 시스템이 이메일 발송 안내 표시: "승인 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
7a. 시스템이 해당 참가자에게 승인 알림 이메일 발송
7a. 관리자가 확인 클릭
8a. 참가자에게 승인 알림 이메일 발송

**거절 흐름(거절 선택 시)**:
4b. 사유 입력(선택)
5b. 시스템이 이메일 발송 안내 표시: "거절 알림 이메일이 [학번]@ksa.hs.kr로 발송됩니다."
7b. 시스템이 해당 참가자에게 거절 알림 이메일 발송
6b. 관리자가 확인 클릭
7b. 신청 상태가 "거절"로 변경됨
8b. 참가자에게 거절 알림 이메일 발송

**종료 조건(Postconditions)**:
- 승인 시: 참가자의 상태가 "승인"으로 변경
- 거절 시: 상태가 "거절"로 변경

### UC-003: 번역 과제 생성 (Create Translation Task)

**행위자(Actor)**: 관리자

**관련 사용자 스토리**:
> US-005: **관리자**로서, 새로운 번역 과제를 생성하고 **싶다**, 참가자에게 번역 작업을 할당하기 **위해서**
> US-006: **관리자**로서, 가온누리 게시글 과제와 외부 게시글 과제를 구분하고 **싶다**, 과제의 성격을 명확히 하기 **위해서**
> US-033: **관리자**로서, 과제를 특정 번역버디에게 수동으로 할당하고 **싶다**, 특별한 사정이나 요청이 있을 때 대응하기 **위해서**

**전제 조건(Preconditions)**:
- 시스템이 번역 기간(Phase 2) 상태

**기본 흐름(Main Flow)**:
1. 관리자가 과제 유형 선택: 가온누리 게시글 또는 외부 게시글
2. 관리자가 게시글 유형 및 Work ID 입력
	- 가온누리 게시글: 문자열 ID (게시글 번호)
	- 외부 게시글: 과제명 (최대 50자)
3. 시스템이 유형 및 Work ID 중복 여부 확인
4. [중복 시] 오류 메시지 표시 후 종료
5. 관리자가 배정 방식 선택: **자동 배정** 또는 **수동 배정**

**자동 배정 흐름(자동 배정 선택 시)**:
6a. 시스템이 순환 알고리즘으로 할당 대상 번역버디 미리보기 표시
	- 휴식 상태 번역버디 제외
	- 순환(라운드 로빈) 방식으로 할당
	- 동시에 여러 관리자가 과제 생성 시 경합 발생 가능 (DB 트랜잭션으로 처리)
7a. 관리자가 배정 대상 확인 후 과제 생성 요청
8a. 시스템이 과제를 선택된 번역버디에게 할당
9a. 할당 시각 기록

**수동 배정 흐름(수동 배정 선택 시)**:
6b. 관리자가 드롭다운에서 할당할 번역버디 선택
	- 휴식 상태 번역버디는 선택지에서 제외

### UC-021: 48시간 제출 리마인드 (48-Hour Submission Reminder)

**기본 흐름(Main Flow)**:

3. 시스템이 이메일 수신 여부 확인
4. [수신 허용 시] 해당 번역버디에게 리마인드 이메일 발송: "과제가 할당된 지 48시간이 경과되었습니다. 제출해 주세요."

**종료 조건(Postconditions)**:
- 해당 번역버디에게 리마인드 이메일 발송됨 (수신 허용 시)
- 모든 관리자에게 리마인드 알림 이메일 발송됨
- 리마인드 발송 기록이 저장됨

## 6. 비즈니스 규칙 (Business Rules)

### 6.1 사용자 관리
- BR-001: 하나의 학번은 중복 참가 불가
- BR-002: 이름은 2~4글자 허용
- BR-003: 학번 형식은 XX-XXX (두 자리 연도, 세 자리 번호)
- BR-004: 승인되지 않은 사용자는 시스템 이용 불가

### 6.2 과제 관리
- BR-005: 번역 기간(Phase 2)에만 과제 생성 가능
- BR-006: Work ID는 중복 불가
- BR-007: 가온누리 게시글 과제 ID는 정합한 텍스트 최대 50자 허용
- BR-008: 외부 게시글 과제 ID는 정합한 텍스트 최대 50자 허용
- BR-047: 과제 배정 방식은 '자동 배정'과 '수동 배정' 중 선택 가능
- BR-048: 자동 배정 시 휴식 상태가 아닌 번역버디를 대상으로 순환(라운드 로빈) 방식으로 순차 할당
- BR-049: 수동 배정 시 관리자가 특정 번역버디를 직접 지정 (휴식 상태 제외)
- BR-050: 동시 다발적 과제 생성 시 DB 트랜잭션 잠금으로 경합 처리
### 6.3 시간 관리
- BR-012: 1글자 = 3.942초 (공백 제외)
- BR-013: 지각 기준은 할당 후 48시간(172,800초) 초과
- BR-014: 지각 시 경고 발행, 봉사시간은 유지

### 6.5 접근 제어
- BR-015: 비회원은 접근 불가
- BR-016: 참가자는 개인 정보, 과제, 사전 조회 가능
- BR-017: 관리자는 전체 관리 권한 보유
- BR-018: 웹 인터페이스는 인증 토큰 기반 접근
- BR-029: 카카오톡 챗봇은 대화 기반 접근 (로그인 개념 없음)

### 6.6 기간 제어
- BR-019: 기간 순서는 비활성(0) -> 모집(1) -> 번역(2) -> 정산(3) -> 비활성(0) 순으로 순환
- BR-020: 정산 기간에는 신규 과제 생성 불가
- BR-021: 비활성 기간에는 대부분의 사용자 기능 중단 (기간 전환만 제외)

### 6.7 이메일 알림
- BR-019: 이메일 주소 형식은 [학번]@ksa.hs.kr로 자동 간주
- BR-020: 과제 할당 시 해당 번역버디에게 이메일 발송
- BR-021: 신청 승인/거절 시 해당 참가자에게 이메일 발송
- BR-022: 경고 발행 시 해당 번역버디에게 이메일 발송
- BR-023: 관리자 승급/강등 시 해당 사용자에게 이메일 발송
- BR-024: 경고 발행, 관리자 승급/강등 실행 전 이메일 발송 안내 표시 및 확인 필수
- BR-025: 이메일 발송 실패 시 관리자에게 오류 메시지 표시

### 6.7 번역 사전
- BR-043: 한국어 용어는 최대 200자 허용
- BR-044: 다대일 매핑 가능: 여러 한국어 용어가 한 영어 단어로 매핑됨
- BR-045: 일대다 매핑 가능: 한 한국어 용어가 여러 영어 단어로 매핑됨
- BR-046: 사전 룰은 자유롭게 편집 가능 (추가, 삭제, 수정)

### 6.8 이메일 수신 설정
- BR-032: 이메일 수신 여부는 사용자가 직접 설정 가능
- BR-033: 이메일 하단에 "수신거부" 링크 포함
- BR-034: "수신거부" 링크 클릭 시 인증 토큰 또는 이메일_인증_비밀키를 통해 본인 확인 필요
	- 관리자: 인증 토큰 사용 (기존 방식)
	- 일반 번역버디: 이메일_인증_비밀키 사용 (비밀키는 사용자별 고유 ID)
- BR-035: 수신거부한 사용자에게는 이메일 발송 금지
- BR-035: 수신거부한 사용자에게는 이메일 발송 금지
- BR-036: 수신거부 사용자에게 메일 발송 시도 시 관리자에게 경고 메시지 표시

### 6.9 인증 토큰 관리
- BR-037: 인증 토큰은 관리자에게만 발급됨
- BR-038: 인증 토큰은 카카오톡 챗봇을 통해서만 발급/재발급 가능
- BR-039: 인증 토큰 유효기간은 발급일 포함 6개월
- BR-040: 토큰 만료 시 재발급 요구 메시지 표시
- BR-041: 토큰 재발급 시 기존 토큰은 자동 무효화됨
- BR-042: 웹 인터페이스 접근은 유효한 인증 토큰 필수

## 9. 용어 사전 (Glossary)

| 용어 | 설명 |
|------|------|
| 번역버디(Translation Buddy) | 봉사활동에 참여하는 학생 |
| 가온누리(Gaonnuri) | 학교 웹사이트 공지 시스템 |
| Work ID | 과제 고유 식별자 |
| 보정(Adjustment) | 번역 글자 수에 추가/차감되는 보정 자수. 봉사시간은 파생 필드로 자동 계산됨 |
| 휴식(Rest) 상태 | 과제 할당에서 일시 제외된 번역버디 |
| 지각 | 과제 할당 후 48시간 경과 후 제출 |
| Phase 0, 1, 2, 3 | 각각 비활성, 모집, 번역, 정산 기간 |
| 이메일 알림 | 중요 이벤트 발생 시 참가자에게 자동 발송되는 이메일 |
| 이메일 수신 거부 | 사용자가 이메일 수신을 거부한 상태 |
| 인증 토큰(Authentication Token) | 웹 인터페이스 접근을 위한 관리자 인증 수단 |
| 카카오톡 챗봇 | 번역버디가 사용하는 메인 인터페이스, 대화 기반 |
| 웹 인터페이스 | 관리자만 사용하는 인터페이스, 토큰 기반 접근 |
